WIRESHARK

Handheld fishing simulator

WIRESHARK is a handheld fishing simulator/game intended to replicate the functionality and gameplay of the Stardaw Valley fishing minigame.

Users can navigate menus, select the fishing gear they would like to use, and catch a fish. Gameplay works in two "phases":
1. The user "casts" a bobber out into a body of water and waits for a catch. To start a cast, the user holds down the green button on the handle, swings the device, and releases the button to throw the line. The distance of a cast is dependent on the length and speed of a swing.
2. When a fish is caught, it enters a "reeling screen", in which the player uses the reel to move their reel bar up or down the screen. Keeping the fish within the reel bar for 10s will result in a catch. If the fish leaves the zone for too long, the fish escapes.

Features and Mechanics

To control the device, players use the reel to move their selection up or down, and use the green button mounted on the handle to select.
Below is a list of features included in WIRESHARK:

Title Screen

From the title screen, users can select one of three previously saved games, loaded in from AWS.

New User Creation

If a new save is selected, users are redirected to the New User screen, where they can use the reel to scroll through letters, with the green button to select.
(One press: choose letter, Double press: confirm name)

Stats Screen

The stats screen holds the current progress of a user, including their name, total fish caught, coins, and images for their active rod and lure. From here, the user can select which menu to display on their second screen:
(Cast, Poles, Lures, Shop, Fish Almanac, Save and Exit)

Pole Selection

From the pole selection screen, users can choose a new rod to select as their active rod, which will update on the stats screen upon selection. When hovering over a rod, the rod's name and speed are displayed.

Lure Selection

Much like the pole screen, the lure screen allows users to navigate to which lure they would like to use.

Fish Almanac

The fish almanac displays information about each type of fish found. An undiscovered fish will simply show "???". Regardless of whether the fish has been found, this screen informs the user of that fish's favorite lure, giving the player insight into how it can be caught.

Casting

The casting screen is the initial entry point for gameplay. Users are shown a solid blue screen representing a lake. The bobber's distance on the screen depends on the distance they've cast.

Reeling

On this screen, players engage in a reeling mini-game where timing and precision are key as they battle with the fish.

Design and Implementation

Below are the design and implementation details split into hardware and software components.

Hardware

Intended to replicate the look and feel of a fishing rod, our hardware componenets are as follows:
TI CC3200: Built around the TI CC3200 development board, WireShark’s core processing is handled by this microcontroller.
Battery: The system is powered by a Lithium-Ion cell battery (3.3V).
Frame: The electronics are mounted on a piece of wood and attached to a sawed-off golf club, for a design similar to a fishing pole.
OLED Screens: Two separate Adafruit OLED screens are used—Screen 1 for menu selection and Screen 2 for gameplay and submenu options. They communicate over SPI/GPIO and share most signal lines, with a modified OLEDCS line to select the desired chip.
Tactile Switch: A tactile switch is attached to the back of the golf-club handle, connected to the CC3200 via GPIO. This active-high button lets users confirm selections on the displays.
Rotary Encoder: A KY-040 rotary encoder is connected between the microcontroller and a reel handle. It provides directional input by reading two GPIO pins simultaneously—one going low slightly before the other to indicate rotation direction.
Mini-Speaker/DFPlayer: An Adafruit mini-speaker and a DFPlayer module (communicating via UART) enable playback of MP3 files for music and sound effects.

Software

The software portion of our project was split into several different files to allow for modularity. Here are the files and a brief description of each:
main.c: Calls all initializations and enters the infinite loop. In the infinite loop, checks for changes in the selection or state, which then calls the respective function in state_machine.c
interrupt_handler.c: Initializes interrupts and stores the current selection. Updates a state flag if the state has changed and a screen_updating flag to temporarily disable interrupts while the screen is updating.
state_machine.c: Called from main and calls functions to update the state or selection in the screen1 and screen2 files, depending on the current state of each screen and the current active screen.
saved_data.c: Stores player data for 3 different saves and holds the data for the current active player. Upon saves or loads, the data is sent or loaded from AWS. The current active player can be switched from the title screen (screen1)
screen1.c: Holds a function to deal with state and selection changes in screen 1. Holds two functions per screen: Setup and Update. When the state changes, a new screen setup is called, when the selection or an item is changed, it calls the respective update function. Screen1 has the following states: Title Screen, New User Screen, Stats Screen.
screen2.c: Same functionality as screen1. Screen2 states: Poles Screen, Lures Screen, Fish Almanac Screen, Game Screen. When the screen is not in use (when active screen = 1), the screen is filled with black. When cast is selected from the stats screen, screen2.c calls game.c to handle the game.
game.c: Manages the main gameplay. Polls sensors and accelerometer to determine cast distance, then sets up the reeling screen and runs the reeling screen. Harder fish move faster, better poles move faster.
fish.c, poles.c, lures.c: Store data for their respective object (names, images, speed, etc).
We also heavily modified the Adafruit libraries to suit our needs, including drawing bitmaps, resetting and handling two screens in parallel, and screen buffering.
